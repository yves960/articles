# 每日工作流

本文档定义了团队的日常工作流程。

---

## 📅 每日任务时间表

| 时间 | 任务 | 执行者 | 内容 |
|------|------|--------|------|
| **09:00** | 早间汇报 | 蔚 | 创意汇报 + 产品优化情况 |
| **10:00-18:00** | 产品体验 | 金克丝 | 体验 ~/Projects/self 下的项目 |
| **10:00-18:00** | 创意发现 | 金克丝+蔚 | 看网上产品，讨论创意 |
| **14:00** | 优化建议 | 金克丝→艾克 | 提交优化建议 |
| **15:00-17:00** | 代码修改 | 艾克 | 实现优化 |
| **17:00-19:00** | 测试验证 | 凯特琳 | 测试修改 |
| **21:00** | 晚间汇报 | 蔚 | 产品优化情况 |

---

## 🔄 工作流 1：产品体验与优化

### 触发：每天 10:00

### 流程：

```
1. 金克丝体验项目
   ├─ 随机选择 ~/Projects/self 下的一个项目
   ├─ 使用产品，记录体验
   └─ 输出：优化建议列表

2. 金克丝 → 艾克
   ├─ 发送优化建议给艾克
   └─ 艾克评估可行性

3. 艾克实现修改
   ├─ 选择优先级最高的建议
   ├─ 编写代码
   └─ 输出：修改内容

4. 艾克 → 凯特琳
   ├─ 发送修改内容给凯特琳
   └─ 凯特琳测试验证

5. 凯特琳反馈
   ├─ 测试通过 → 记录到优化日志
   └─ 测试失败 → 返回艾克修改
```

### 输出文件：
- `memory/optimization-log.md` - 优化日志
- `memory/suggestions/YYYY-MM-DD.md` - 每日建议

---

## 🔄 工作流 2：创意发现

### 触发：每天 14:00

### 流程：

```
1. 金克丝发现产品
   ├─ 浏览 Product Hunt / TechCrunch / GitHub Trending
   ├─ 选择1-2个有趣的产品
   └─ 输出：产品分析

2. 金克丝 + 蔚讨论
   ├─ 金克丝分享发现
   ├─ 蔚给出意见
   └─ 输出：创意点子

3. 记录创意
   └─ 写入 memory/ideas/YYYY-MM-DD.md
```

### 输出文件：
- `memory/ideas/YYYY-MM-DD.md` - 每日创意

---

## 📢 工作流 3：汇报

### 早间汇报 (09:00)

发送给用户的内容：
```
🌅 早间汇报 - YYYY-MM-DD

## 💡 昨日创意
[金克丝和蔚发现的产品创意]

## 📊 产品优化进度
[各项目优化情况]

## 📋 今日计划
[今天要做的优化]
```

### 晚间汇报 (21:00)

发送给用户的内容：
```
🌙 晚间汇报 - YYYY-MM-DD

## ✅ 今日完成
[完成的优化]

## 🔄 进行中
[正在进行的优化]

## ❌ 待解决问题
[发现的问题]
```
