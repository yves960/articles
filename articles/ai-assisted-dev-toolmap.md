# AI辅助研发工具地图

> 以Agent为核心，Harness为边界，覆盖从设计到部署的完整研发流程

---

## 核心架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              研发流程（横向）                                  │
│                                                                              │
│    需求分析  →  设计  →  开发  →  测试  →  部署  →  运维                      │
│                                                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                        反馈层 (Feedback Layer)                       │    │
│  │  验证 · 检查 · 纠正 · 评估                                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                        执行层 (Execution Layer)                       │    │
│  │  工具 · 命令 · 操作 · 集成                                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                        信息层 (Information Layer)                     │    │
│  │  知识 · 上下文 · 规范 · 历史                                          │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│                              ┌─────────┐                                    │
│                              │  Agent  │                                    │
│                              │  Core   │                                    │
│                              └─────────┘                                    │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 一、信息层 (Information Layer)

**职责**：为Agent提供知识、上下文、规范和历史信息

### 1.1 项目知识

| 类型 | 内容 | 工具/实现 |
|------|------|----------|
| **代码库索引** | 代码结构、依赖关系、调用链 | Codebase Indexing, AST分析 |
| **API文档** | 接口定义、参数、响应格式 | OpenAPI/Swagger, API文档 |
| **架构文档** | 系统架构、模块划分、技术栈 | ADR, 架构图 |
| **数据库Schema** | 表结构、关系、索引 | ER图, Migration文件 |

**工具示例**：
- `codebase-indexer`: 代码库向量化索引
- `ast-analyzer`: 代码结构分析
- `dependency-graph`: 依赖关系图
- `api-docs-parser`: API文档解析

### 1.2 业务上下文

| 类型 | 内容 | 工具/实现 |
|------|------|----------|
| **需求文档** | 用户故事、功能需求、验收标准 | PRD, User Stories |
| **设计文档** | 详细设计、接口设计、数据设计 | 设计稿, Figma |
| **术语表** | 业务术语、领域概念 | Glossary, DDD模型 |
| **流程图** | 业务流程、状态机 | Flowchart, BPMN |

**工具示例**：
- `requirement-parser`: 需求文档解析
- `figma-to-spec`: 设计稿转规范
- `glossary-manager`: 术语表管理

### 1.3 规范约束

| 类型 | 内容 | 工具/实现 |
|------|------|----------|
| **代码规范** | 命名、格式、最佳实践 | ESLint, Prettier, Checkstyle |
| **架构规范** | 分层、模块化、依赖规则 | ArchUnit,依赖分析 |
| **安全规范** | 敏感数据处理、权限控制 | 安全扫描规则 |
| **API规范** | RESTful规范、版本控制 | API Linter |

**工具示例**：
- `spec-validator`: SPEC验证器
- `architecture-linter`: 架构规范检查
- `security-rules`: 安全规则引擎

### 1.4 历史信息

| 类型 | 内容 | 工具/实现 |
|------|------|----------|
| **Git历史** | 提交记录、变更历史 | Git Log, Blame |
| **Issue/PR** | 问题讨论、代码审查 | GitHub/GitLab API |
| **运维记录** | 故障历史、变更记录 | Incident Log |
| **用户反馈** | Bug报告、功能请求 | Issue Tracker |

**工具示例**：
- `git-analyzer`: Git历史分析
- `pr-summarizer`: PR摘要生成
- `incident-learner`: 故障学习

### 1.5 外部知识

| 类型 | 内容 | 工具/实现 |
|------|------|----------|
| **技术文档** | 框架文档、API参考 | 官方文档、Stack Overflow |
| **最佳实践** | 设计模式、架构模式 | 技术博客、开源项目 |
| **领域知识** | 行业标准、合规要求 | 领域专家知识 |

**工具示例**：
- `web-search`: 网络搜索
- `doc-retriever`: 文档检索
- `knowledge-base`: 知识库

## 二、执行层 (Execution Layer)

**职责**：为Agent提供工具、命令、操作和集成能力

### 2.1 代码操作

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **文件读写** | 读取、创建、修改、删除文件 | file-read, file-write, file-edit |
| **代码生成** | 根据需求生成代码 | code-generator, template-engine |
| **代码重构** | 重命名、提取方法、移动文件 | refactoring-tools |
| **代码搜索** | 关键词搜索、语义搜索 | grep, semantic-search |

**工具示例**：
- `read/write/edit`: 文件操作
- `code-generator`: 代码生成
- `refactor-agent`: 重构Agent
- `semantic-search`: 语义搜索

### 2.2 开发工具

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **Shell命令** | 执行任意命令 | bash, shell-exec |
| **包管理** | 安装、更新、删除依赖 | npm, pip, maven |
| **构建工具** | 编译、打包、构建 | webpack, maven, gradle |
| **版本控制** | Git操作 | git-commit, git-push, git-merge |

**工具示例**：
- `bash`: Shell命令执行
- `package-manager`: 包管理器
- `build-runner`: 构建运行器
- `git-tools`: Git工具集

### 2.3 测试工具

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **单元测试** | 运行测试、生成测试 | jest, pytest, junit |
| **集成测试** | API测试、端到端测试 | postman, cypress |
| **覆盖率** | 代码覆盖率分析 | coverage-tools |
| **性能测试** | 负载测试、基准测试 | jmeter, k6 |

**工具示例**：
- `test-runner`: 测试运行器
- `test-generator`: 测试生成
- `coverage-analyzer`: 覆盖率分析
- `performance-tester`: 性能测试

### 2.4 部署工具

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **容器化** | Docker构建、推送 | docker-build, docker-push |
| **K8s部署** | 部署、扩缩容、回滚 | kubectl, helm |
| **CI/CD** | 流水线触发、状态查询 | Jenkins, GitHub Actions |
| **配置管理** | 配置更新、密钥管理 | config-server, vault |

**工具示例**：
- `docker-tools`: Docker工具
- `k8s-tools`: Kubernetes工具
- `pipeline-trigger`: 流水线触发
- `config-manager`: 配置管理

### 2.5 运维工具

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **监控查询** | 日志查询、指标查询 | prometheus, grafana, elk |
| **故障排查** | 链路追踪、根因分析 | jaeger, zipkin |
| **运维操作** | 重启、扩缩容、回滚 | runbook-automation |
| **告警处理** | 告警分析、自动响应 | alert-manager |

**工具示例**：
- `log-query`: 日志查询
- `metrics-query`: 指标查询
- `trace-analyzer`: 链路分析
- `runbook-executor`: 运维手册执行

### 2.6 外部集成

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **数据库** | 查询、迁移、备份 | sql-executor, migration-runner |
| **消息队列** | 发送、消费消息 | kafka-tools, rabbitmq-tools |
| **API调用** | HTTP请求、RPC调用 | http-client, rpc-client |
| **第三方服务** | 外部服务集成 | openapi-client, sdk |

**工具示例**：
- `db-tools`: 数据库工具
- `mq-tools`: 消息队列工具
- `http-client`: HTTP客户端
- `external-api`: 外部API集成

## 三、反馈层 (Feedback Layer)

**职责**：验证、检查、纠正和评估Agent的行为

### 3.1 静态检查

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **代码风格** | Lint检查、格式化 | ESLint, Prettier, Checkstyle |
| **类型检查** | 类型错误检测 | TypeScript, mypy |
| **安全扫描** | 漏洞检测、敏感信息 | SonarQube, Snyk |
| **依赖检查** | 依赖漏洞、许可证 | dependabot, license-checker |

**工具示例**：
- `linter`: 代码风格检查
- `type-checker`: 类型检查
- `security-scanner`: 安全扫描
- `dependency-checker`: 依赖检查

### 3.2 动态验证

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **单元测试** | 测试通过率、覆盖率 | test-runner |
| **集成测试** | API测试、端到端测试 | postman, cypress |
| **契约测试** | API契约验证 | pact, spring-cloud-contract |
| **回归测试** | 功能回归验证 | test-automation |

**工具示例**：
- `test-validator`: 测试验证
- `contract-tester`: 契约测试
- `regression-runner`: 回归测试

### 3.3 代码审查

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **自动审查** | 代码质量、最佳实践 | code-review-agent |
| **差异分析** | 变更影响、风险分析 | diff-analyzer |
| **安全审查** | 权限、敏感数据处理 | security-review |
| **性能审查** | 性能问题、优化建议 | performance-review |

**工具示例**：
- `auto-reviewer`: 自动审查
- `diff-analyzer`: 差异分析
- `security-reviewer`: 安全审查
- `performance-reviewer`: 性能审查

### 3.4 架构验证

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **依赖检查** | 循环依赖、分层违规 | archunit, dependency-check |
| **接口兼容** | API兼容性检查 | api-compat-checker |
| **配置验证** | 配置正确性检查 | config-validator |
| **架构演进** | 架构债务追踪 | tech-debt-tracker |

**工具示例**：
- `architecture-validator`: 架构验证
- `compat-checker`: 兼容性检查
- `config-validator`: 配置验证

### 3.5 运行验证

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **健康检查** | 服务健康状态 | health-check |
| **冒烟测试** | 部署后验证 | smoke-test |
| **灰度验证** | 金丝雀发布验证 | canary-validator |
| **回滚检测** | 异常检测、自动回滚 | rollback-detector |

**工具示例**：
- `health-checker`: 健康检查
- `smoke-tester`: 冒烟测试
- `canary-validator`: 灰度验证
- `auto-rollback`: 自动回滚

### 3.6 反馈闭环

| 类型 | 能力 | 工具/实现 |
|------|------|----------|
| **错误学习** | 错误模式记录、避免 | error-learner |
| **规范更新** | 根据反馈更新规范 | spec-updater |
| **知识积累** | 经验沉淀、知识库 | knowledge-accumulator |
| **能力进化** | 根据反馈优化能力 | capability-evolver |

**工具示例**：
- `feedback-collector`: 反馈收集
- `lesson-learner`: 教训学习
- `knowledge-updater`: 知识更新
- `capability-optimizer`: 能力优化

## 四、研发流程覆盖

### 4.1 需求分析阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 需求理解 | 需求文档、术语表 | 需求解析、问题澄清 | 需求完整性检查 |
| 影响分析 | 代码库索引、架构文档 | 代码搜索、依赖分析 | 影响范围评估 |
| 方案设计 | 设计文档、最佳实践 | 方案生成、技术选型 | 方案评审 |

**Agent能力**：
- 需求理解和澄清
- 影响范围分析
- 技术方案建议

### 4.2 设计阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 架构设计 | 架构规范、最佳实践 | 架构图生成、ADR编写 | 架构评审 |
| 接口设计 | API规范、数据模型 | OpenAPI生成、接口Mock | 契约验证 |
| 数据设计 | 数据库Schema、ER图 | DDL生成、迁移脚本 | 数据一致性检查 |

**Agent能力**：
- 架构设计辅助
- API设计生成
- 数据库设计

### 4.3 开发阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 代码编写 | 代码规范、模板代码 | 代码生成、文件操作 | Lint检查、类型检查 |
| 单元测试 | 测试规范、覆盖率要求 | 测试生成、测试运行 | 覆盖率验证 |
| 代码审查 | 审查规范、最佳实践 | 自动审查、差异分析 | 审查反馈 |

**Agent能力**：
- 代码生成和修改
- 测试用例生成
- 代码自动审查

### 4.4 测试阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 集成测试 | 测试环境、测试数据 | 测试执行、数据准备 | 测试结果分析 |
| 性能测试 | 性能基准、测试场景 | 性能测试执行 | 性能报告 |
| 安全测试 | 安全规范、漏洞库 | 安全扫描、渗透测试 | 安全报告 |

**Agent能力**：
- 测试执行和报告
- 性能测试分析
- 安全漏洞扫描

### 4.5 部署阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 构建打包 | 构建配置、依赖列表 | 构建、打包、镜像推送 | 构建验证 |
| 环境部署 | 部署配置、环境变量 | K8s部署、配置注入 | 部署验证 |
| 发布验证 | 发布清单、回滚预案 | 冒烟测试、灰度验证 | 发布状态检查 |

**Agent能力**：
- 构建和打包
- 部署执行
- 发布验证

### 4.6 运维阶段

| 阶段任务 | 信息层 | 执行层 | 反馈层 |
|----------|--------|--------|--------|
| 监控告警 | 监控配置、告警规则 | 指标查询、日志检索 | 异常检测 |
| 故障处理 | 运维手册、历史故障 | 故障诊断、自动修复 | 处理效果验证 |
| 持续优化 | 性能基线、优化建议 | 性能调优、资源调整 | 优化效果评估 |

**Agent能力**：
- 监控和告警分析
- 故障诊断和处理
- 性能优化建议
